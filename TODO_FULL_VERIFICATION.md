# Полная проверка программы (до-сканальной и сквозной)

Этот файл — чек‑лист для тщательной валидации функциональности генератора ГНСС: от парсинга RINEX до формирования навигационных сообщений и IF‑сэмплов. Выполняется в несколько фаз. Все команды запускать из корня репозитория.

## 0) Подготовка окружения
- Инструменты: `rustc 1.7x+`, `cargo`, `rg`, `ffmpeg` (необязательно для аудио), `gnuplot` (для спектров — опционно).
- Фичи сборки:
  - CPU: `cargo build --release` (по умолчанию SSE/AVX CPU).
  - AVX‑512: `cargo build --release --features avx512` (если CPU поддерживает AVX‑512).
  - GPU/CUDA 12.5: `cargo build --release --features gpu` (если установлен CUDA 12.5).
- Данные:
  - Папки RINEX/Eph: `Rinex_Data/`, `EphData/` (RO).
  - Пресеты: `presets/*.json`.
  - Выходы: `generated_files/` (игнорируется Git) — сюда писать тяжелые артефакты.
- Быстрая проверка сборки: `cargo check`, затем `cargo test` (все тесты должны пройти).

## 1) Смоук‑тесты (быстро, <1 мин)
- Юнит‑тесты математики/кодеков/навигации: `cargo test` — ОК.
- Интеграционные тесты формирования навсообщений: `cargo test nav_message_formation` — ОК.
- Быстрый ран без больших данных: `cargo run -- presets/GPS_L1_only.json` — должен завершиться без паник; создать файл в `generated_files/`.

## 2) Парсинг и нормализация RINEX (2.x/3.x)
Проверить, что из RINEX извлекаются «родные» эфемериды всех систем без конвертаций:
- GPS: GpsEphemeris (ephemeris/alm/UTC/iono).
- GLONASS: GlonassEphemeris (slot/freq/tb/tk/gamma…), используется напрямую в G/NAV (без GPS→GLO).
- BeiDou: BDS‑2/3 «как есть» (BDT неделя/TOE корректны, сдвиги относительно GPS недели учтены).
- Galileo: GST неделя и временные поля.
- Краевые случаи:
  - Перелом недели/суток (604800/86400), leap seconds.
  - Смесь систем в одном файле, «дырки» по эпохам.
- Команда для отладки: `cargo run --bin spectrum_analyzer -- <выходной_файл> <iq4|iq8>` — для проверки наличия модуляции.

## 3) Формирование навигационных сообщений (бит‑уровень)
Общие проверки для всех систем:
- Плотность битов (не нули), длина кадра/страницы/строки строго по формату.
- CRC/чётность/LDPC считаются и корректно добавляются.
- Расписание сообщений по времени (TOW/страницы/суперкадры) соблюдается.

### 3.1 GPS
- LNAV (L1CA):
  - TLM/HOW (преамбула, D29*/D30*, паритет): длина 10×30 бит/субкадр, 5 субкадров, страницы 4/5 (альманах/UTC/iono/здоровье).
  - Тесты: в интеграционных — LNAV выдаёт непустые 300 бит; дополнительно проверить паритет таблицей.
- CNAV (L2C):
  - MT 10/11/30/31/33/37, CRC24Q и сверточное кодирование, суперкадр 1200 s.
- L5‑CNAV:
  - 300 бит полезных + CRC24Q, интерливинг и свёрточное кодирование; проверка длины 600 бит после FEC.
- CNAV‑2 (L1C):
  - Субкадры 1/2/3 на спутник; кадр 1800 бит. Сейчас «полное по данным» (поля заполнены), строгую ICD‑укладку и LDPC по матрице — вынести в этап 6.

### 3.2 GLONASS (G/NAV)
- Строки 1–4 (эфемериды), 5–15 (альманах/UTC); 85 инфо бит + 8 Хэмминга + 7 паддинга = 100 бит.
- Ввод: только `GlonassEphemeris` из RINEX. Проверить: Bn/P/tb/tk/gamma, компоновка полей, контрольная сумма.

### 3.3 Galileo
- I/NAV (E1/E5b): слова 1–10 (эфемериды/альманах) и 17–20 (RS), CRC24Q + свёртка; чередование even/odd.
- F/NAV (E5a): страничная структура, CRC24Q, FEC; проверка вставки GST/UTC/iono.

### 3.4 BeiDou
- D1/D2 (BDS‑2, B1I/B2I/B3I):
  - Субкадры 1–5, страницы альманаха/UTC/здоровья; BCH и перестановка; исправлено безопасное заполнение страниц здоровья.
- B1C (BCNav1):
  - Субкадры 2/3, CRC+LDPC; итоговый кадр 1800 бит; возврат кода 0 (успех).
- B2a (BCNav2):
  - MT 10/11/30/34; CRC/LDPC; «полное по данным» (строгая ICD — в этап 6).
- B2b (BCNav3):
  - CRC и LDPC; защита от выхода за матрицу.

## 4) Сквозная генерация IF‑данных
- Команда (примеры):
  - `cargo run --release -- presets/GPS_L1_only.json`
  - `GNSS_PARALLEL_MODE=true cargo run --release -- presets/GPS_BDS_GAL_triple_system.json`
- Проверки:
  - PRN‑коды: длительность, автокорреляция/кросс‑корреляция (по выборочным SV), частоты (L1/L2/L5, E1/E5/E6, B1/B2/B3, G1/G2).
  - Навигационные биты действительно накладываются на несущую (смотрим спектр и переходы знака с интервалами 20/10/1/… мс в зависимости от сигнала).
  - Доплер и фаза: градиенты фазы, отсутствие скачков при смене навбита.
  - Амплитуды: соответствуют заданным в пресете (CN0/PowerControl).
  - Файл анализатора: `cargo run --bin spectrum_analyzer -- <путь> <iq4|iq8>` — нет клиппинга, видим боковые полосы.

## 5) Время/системы координат
- Временные шкалы: GPS/GST/BDT/GLONASS/UTC — конверсии, переломы недель/суток, leap seconds.
- Координаты: точность `ecef↔lla` ~ 1 см; вектор ENU, курсы/скорости (см. юнит‑тесты).

## 6) Строгая ICD‑валидация (расширяемая)
Отдельный этап (по мере необходимости):
- CNAV‑2 (L1C): точная укладка MT 10/11/15/30–37, «настоящий» LDPC по ген. матрице; побитовые эталоны.
- BeiDou B2a: строгая упаковка эпhemeris/clock/alm/UTC/iono, LDPC по матрице.
- LNAV паритет: побитовая сверка нескольких подкадров с эталоном.
- F/NAV/I/NAV: проверки CRC24Q и расписаний страниц (временные маски).

## 7) Производительность и устойчивость
- Серии прогонов (CPU / AVX‑512 / GPU), для 10–60 s:
  - Метрики: samples/sec, x realtime, загрузка CPU/GPU, память.
  - Для AVX‑512: `--features avx512`; для GPU: `--features gpu`.
- Многосистемные профили: пресеты `GPS_BDS_GAL_triple_system.json`, частоты 5/18/22/24 MHz.
- Протоколировать время в `full_output.log` (не коммитить).

## 8) CI/статическая проверка
- Формат/линт: `cargo fmt --all && cargo clippy -- -D warnings`.
- Тесты: `cargo test`; интеграционные: `cargo test nav_message_formation`.
- Проверка отсутствия больших файлов в Git: выходные в `generated_files/`, логи/графики не коммитить.

## 9) Отчёт по проверке
- Для каждого пресета:
  - Команда запуска, длительность, параметры (частота дискретизации, системы).
  - Скриншот/график спектра (опционально, в корень или локально без коммита).
  - Статистика: скорость/память/количество спутников/ошибки CRC/паритета (0).
- Сводная таблица по системам, где «Строго ICD»/«Полное по данным»/«Проверено плотностью/CRC».

## 10) Открытые пункты
- Довести CNAV‑2/B2a до строгой ICD‑реализации (кодировщики LDPC, все поля).
- Док‑тесты: вернуть `doctest = true` после разметки примерных блоков как кода.
- Автотесты с RINEX «из мира» (несколько реальных файлов разных агентств и дат).
